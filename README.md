# 🍱 公司內部點餐系統

這是一套純前端實作的公司內部點餐工具，無須架設後端伺服器！完全依賴 **Google OAuth** 進行身分驗證，並使用 **Google Sheets 試算表** 作為資料庫。適合公司內部快速導入、輕鬆管理每日員工點餐。

---

## ✨ 最新功能亮點 (v3+)

- **🛒 跨餐廳點餐與購物車預覽**：使用者可以同時從多家不同的餐廳挑選餐點加入購物車，在送出前「一次檢視所有跨餐廳的暫存紀錄」，並可自由新增或刪除購物車內的品項。
- **📋 訂單管理與取消**：在「查詢訂單」頁籤中會清晰依餐廳分組顯示點餐結果，使用者若點錯，可隨時「取消（刪除）自己的訂單」並重新下單。
- **⚙️ 前台直接管理餐廳**：管理員不需再親自進去 Google Sheets 修改菜單，現在直接在介面上的「設定餐廳」分頁，即可輕鬆完成**新增餐廳/品項**與**刪除餐廳**。

## 🍽️ 詳細功能簡介

### 👨‍💻 1. 使用者操作的功能特性（一般成員）
- **身分驗證**：使用 Google Identity Services 進行登入驗證。
- **瀏覽與客製點餐**：瀏覽「今日開放餐廳」與對應菜單，依分類查看餐點，並可自由調整數量（＋／－），以及填寫餐點備註（如口味、客製需求）。
- **跨餐廳購物車功能**：將餐點加入購物車（可多次累加、跨餐廳選購）。送出前可即時檢視購物車內容與總金額，並自由編輯或移除購物車內項目。
- **訂單送出與管理**：確認並送出訂單（由系統自動寫入 Google Sheets API）。送出後可至清單查詢「本人」當日訂單紀錄；若有點錯，允許取消（刪除）自己已送出的訂單。

### 👑 2. 管理者操作的功能特性（管理員）
- **基礎權限**：具備所有「一般成員」點餐與查詢等操作功能。
- **全域訂單查詢**：查詢「所有使用者」的當日跨餐廳訂單。
- **訂單管理權限**：擁有刪除任意一筆訂單的權限（用於修正錯誤）。
- **一鍵匯出**：一鍵複製當日所有訂單（自動產生排版過的格式化文字，用於方便貼至 LINE 等通訊工具向店家叫餐）。
- **前台設定餐廳與菜單**：可從菜單資料中動態載入餐廳清單進行勾選；也能直接在前台執行「新增合作餐廳」、「刪除既有餐廳」的操作。
- **管理整體點餐流程（開放 → 收單 → 匯出 → 清空）**：
  - **開放**：登入系統設定「今日開放餐廳」（控制當日可點餐範圍），並儲存開放給員工。
  - **收單**：結單後，管理員檢視當日送出之所有訂單與總金額。
  - **匯出**：使用「一鍵複製」功能，快速轉貼並完成叫餐。
  - **清空**：確認叫餐完成後，執行「清空當日所有訂單資料」，重置 Orders 試算表，保持後續系統狀態乾淨。

## ⚙️ 使用前必填設定 (重要)

開啟 `all.js`，找到頂部的 `CONFIG` 區塊，填入您自己的 Google 憑證：

```javascript
const CONFIG = {
  // 1. 替換成您的 Google OAuth Client ID
  CLIENT_ID: 'YOUR_CLIENT_ID.apps.googleusercontent.com', 

  // 2. 替換成您的 Google Sheets ID
  SPREADSHEET_ID: 'YOUR_SPREADSHEET_ID',                       
  ...
};
```

### 🔑 CLIENT_ID 取得方式：
1. 前往 [Google Cloud Console](https://console.cloud.google.com)。
2. 建立或選取一個專案，前往「**API 和服務 → 憑證**」。
3. 點擊「**建立憑證 → OAuth 2.0 用戶端 ID**」。
4. 應用程式類型選「**網頁應用程式**」。
5. 在「**已授權的 JavaScript 來源**」加入部署此工具的網址（例：`http://127.0.0.1:5500` 或 `https://your-domain.com`）。
6. 將產生的「用戶端 ID」複製貼上。
> ⚠️ **注意**：Google OAuth 不支援 `file://` 協定，測試時請務必使用 Local Server 開啟。

### 📊 SPREADSHEET_ID 取得方式：
從您的 Google 試算表網址中複製中間的那串英數字：
`https://docs.google.com/spreadsheets/d/【這一段就是 SPREADSHEET_ID】/edit`
> 請務必確認該試算表的存取權限，有開放給需要點餐的員工帳號。

## ❓ 常見問題 (FAQ)

**Q：登入後畫面顯示「尚未取得授權」？**
> 代表該 Google 帳號的 Email 沒有建立在 `Users` 工作表中，請通知管理員將您的信箱加入名單。

**Q：畫面一直處於「轉圈圈」載入狀態？**
> 請檢查 `all.js` 裡的 `CLIENT_ID` 以及 `SPREADSHEET_ID` 是否正確填寫。

**Q：為什麼點擊登入按鈕完全沒有反應？**
> 1. 請確認是透過 HTTP 伺服器開啟（不可直接雙擊使用 `file://` 開啟 HTML）。
> 2. 請前往 GCP 後台，確認「已授權的 JavaScript 來源」是否包含您目前使用的網址。

**Q：我的訂單送出成功了，但在「查詢訂單」卻看不到？**
> 請點擊該頁籤的「🔄 刷新」按鈕重新讀取資料。若仍無資料，請管理員檢查 Google Sheets 中的日期格式是否被意外竄改。

**Q：可以透過手機來點餐嗎？**
> 可以！全系統採用「行動裝置優先（Mobile-First）」實作，透過手機瀏覽器開啟能獲得與 App 原生相近的流暢體驗。
