# 🍱 公司內部點餐工具

純前端實作的公司內部點餐系統，使用 Google OAuth 登入、Google Sheets 作為資料庫，無需後端伺服器。

---

## 📁 檔案結構

```
├── index.html   # 頁面結構與 Google API 腳本載入
├── all.css      # 全域樣式（暖色系、行動裝置優先）
├── all.js       # 所有邏輯：身分驗證、API 呼叫、權限判斷、UI 渲染
└── README.md    # 本說明文件
```

---

## ⚙️ 使用前必填設定

開啟 `all.js`，找到頂部的 `CONFIG` 區塊，填入以下兩個值：

```js
const CONFIG = {
  CLIENT_ID:      'YOUR_CLIENT_ID.apps.googleusercontent.com', // ← 替換這裡
  SPREADSHEET_ID: 'YOUR_SPREADSHEET_ID',                       // ← 替換這裡
  ...
};
```

### CLIENT_ID 取得方式

1. 前往 [Google Cloud Console](https://console.cloud.google.com)
2. 建立或選取一個專案
3. 前往「**API 和服務 → 憑證**」
4. 點擊「**建立憑證 → OAuth 2.0 用戶端 ID**」
5. 應用程式類型選「**網頁應用程式**」
6. 在「**已授權的 JavaScript 來源**」加入你部署此工具的網址
   - 本機開發範例：`http://127.0.0.1:5500`（VS Code Live Server 預設）
   - 正式環境範例：`https://your-company.example.com`
7. 建立後複製「用戶端 ID」貼入 `CLIENT_ID`

> ⚠️ Google OAuth **不支援 `file://` 協定**，請務必透過本機伺服器（如 Live Server）或正式網址開啟。

### SPREADSHEET_ID 取得方式

從 Google Sheets 網址中複製：

```
https://docs.google.com/spreadsheets/d/【這一段就是 SPREADSHEET_ID】/edit
```

> 請確認試算表已分享給所有需要使用此工具的 Google 帳號（或設為公司組織內可存取）。

## 🍽️ 功能說明

### 今日點餐

- 讀取 `TodayConfig` 決定今日可點的餐廳
- 切換餐廳頁籤，菜單依「分類」自動分組顯示
- 每道菜可調整數量（+/−），右側即時顯示小計
- 可填寫備註（如：不要辣、少冰）
- 點擊「＋ 加入點餐清單」將品項放入購物車（不立即送出）

### 購物車（即時預覽）

- 加入後在菜單下方即時顯示目前選取的所有項目
- 可點 ✕ 移除單一項目，或點「清空」重選
- 確認無誤後點「**✅ 確認送出訂單**」
- 系統跳出確認對話框（顯示完整品項與合計）
- 再次確認後才寫入 Google Sheets

### 查詢訂單

- **一般成員**：只能看到自己今日的訂單明細
- **管理員**：可看到所有人今日的訂單，並可點 🗑️ 刪除任意一筆

### 一鍵複製（管理員）

- 將今日全部訂單格式化成純文字
- 點擊後複製到剪貼簿，可直接貼到 LINE 群組供確認

### 設定餐廳（管理員）

- 從 `Menu` 工作表中取得所有餐廳名稱
- 勾選今日要開放的餐廳，儲存後立即生效
- 員工的「今日點餐」頁面將只顯示勾選的餐廳

### 管理員專區

- **刪除個別訂單**：在查詢訂單頁籤，每筆訂單右側的 🗑️ 按鈕
- **清空今日訂單**：頁面底部紅色按鈕，清除 `Orders` 工作表中今日所有記錄

## 🔄 操作流程

### 員工每日流程

```
登入 → 今日點餐
     → 選餐廳、挑餐點、調整數量、填備註
     → 加入點餐清單（可多次加入）
     → 確認購物車明細無誤
     → 確認送出訂單
     → 查詢訂單 → 確認自己的訂單已送出
```

### 管理員每日流程

```
【開始前】
  登入 → 設定餐廳 → 勾選今日餐廳 → 儲存

【收單後】
  查詢訂單 → 一鍵複製 → 貼到 LINE 群組確認
  （如有人點錯）→ 點 🗑️ 刪除該筆訂單

【收單結束後（選擇性）】
  清空今日所有點餐記錄 → 等明日再使用
```

## ❓ 常見問題

**Q：登入後顯示「尚未取得授權」**
→ 該 Google 帳號的 Email 不在 `Users` 工作表中，請管理員新增。

**Q：登入後一直轉圈圈**
→ 確認 `CLIENT_ID` 與 `SPREADSHEET_ID` 是否正確填寫。

**Q：點擊登入按鈕沒有反應**
→ 確認是否透過 HTTP 伺服器開啟（不能直接用 `file://` 開啟）。
→ 確認 GCP 「已授權的 JavaScript 來源」包含目前使用的網址。

**Q：訂單送出成功但「查詢訂單」看不到**
→ 點擊「🔄 刷新」按鈕重新讀取。若仍無資料，確認 Google Sheets 試算表的日期格式是否正確。

**Q：可以在手機上使用嗎？**
→ 可以，介面採行動裝置優先設計，手機瀏覽器可正常使用。
